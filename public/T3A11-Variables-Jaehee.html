What about Javascript?

JavaScript uses reserved keyword var to declare a variable. A variable must have a unique name. Can assign a value to a variable using equal to (=) operator when you declare it or before using it.

    Why is that good and bad?

    Good: Because it allows easier reading of lines of code that contains all you need to know, rather than having to constantly check what the content of a variable is.

    Bad: Because it make readability worse. More variables in the code mean more lines of code and more values/names that need to be kept in mind while scrolling through the code. Also the hidden codes due to variables could lead to messy/spaghetti code.
